# SPESIFIKASI KEBUTUHAN PERANGKAT LUNAK

## Aplikasi To-Do List “Tugasku”

## 1. Pendahuluan

### 1.1 Latar Belakang

Dalam kehidupan sehari-hari, mahasiswa sering memiliki banyak aktivitas seperti mengerjakan tugas kuliah, mengikuti rapat organisasi, menghadiri kelas, dan menyelesaikan pekerjaan pribadi. Banyaknya kegiatan tersebut dapat menyebabkan pengguna lupa terhadap tugas yang harus diselesaikan.

Oleh karena itu, dibutuhkan sebuah aplikasi sederhana yang dapat membantu pengguna mencatat, mengatur, dan memantau tugas secara mudah. Aplikasi “Tugasku” dirancang sebagai aplikasi To-Do List berbasis Flutter yang memiliki tampilan sederhana, menarik, dan mudah digunakan.

### 1.2 Tujuan

Tujuan pengembangan aplikasi “Tugasku” adalah:

1. Membantu pengguna mencatat daftar tugas.
2. Memudahkan pengguna mengubah dan menghapus tugas.
3. Memungkinkan pengguna menandai tugas yang telah selesai.
4. Mengimplementasikan konsep Object Oriented Programming dalam aplikasi Flutter.
5. Menghasilkan aplikasi sederhana berbasis Android dalam bentuk APK.

### 1.3 Ruang Lingkup

Aplikasi “Tugasku” memiliki fitur utama berupa pengelolaan tugas harian dengan operasi CRUD, yaitu:

1. Menampilkan daftar tugas.
2. Menambahkan tugas baru.
3. Mengubah data tugas.
4. Menghapus tugas.
5. Menandai tugas sebagai selesai atau belum selesai.

Aplikasi ini menggunakan penyimpanan data sementara atau in-memory data. Database lokal dapat ditambahkan sebagai pengembangan lanjutan.

## 2. Deskripsi Umum Perangkat Lunak

### 2.1 Perspektif Produk

“Tugasku” adalah aplikasi mobile berbasis Android yang dikembangkan menggunakan Flutter dan bahasa pemrograman Dart. Aplikasi ini bersifat stand-alone dan digunakan oleh satu pengguna.

### 2.2 Fungsi Produk

Fungsi utama aplikasi adalah:

1. Mengelola daftar tugas.
2. Menampilkan status tugas.
3. Mengelompokkan tugas berdasarkan prioritas.
4. Memberikan tampilan antarmuka sederhana dengan warna putih dan biru.
5. Membantu pengguna lebih terorganisir dalam menyelesaikan kegiatan.

### 2.3 Karakteristik Pengguna

Target pengguna aplikasi adalah:

1. Mahasiswa.
2. Pelajar.
3. Pengguna umum yang ingin mencatat tugas harian.
4. Pengguna yang membutuhkan aplikasi sederhana untuk manajemen kegiatan.

### 2.4 Batasan Sistem

Batasan aplikasi:

1. Aplikasi hanya berjalan pada perangkat Android.
2. Data belum menggunakan database permanen.
3. Aplikasi digunakan oleh satu pengguna.
4. Tidak membutuhkan koneksi internet.
5. Fitur notifikasi belum menjadi fitur utama.

## 3. Kebutuhan Fungsional

### 3.1 Menampilkan Daftar Tugas

Sistem dapat menampilkan seluruh data tugas yang telah ditambahkan oleh pengguna.

### 3.2 Menambahkan Tugas

Pengguna dapat menambahkan tugas baru dengan data:

1. Judul tugas.
2. Deskripsi tugas.
3. Prioritas tugas.
4. Status tugas.

### 3.3 Mengubah Tugas

Pengguna dapat mengubah data tugas yang sudah dibuat, seperti judul, deskripsi, dan prioritas.

### 3.4 Menghapus Tugas

Pengguna dapat menghapus tugas yang tidak diperlukan.

### 3.5 Menandai Tugas Selesai

Pengguna dapat mengubah status tugas menjadi selesai atau belum selesai.

### 3.6 Menampilkan Detail Tugas

Pengguna dapat melihat detail tugas yang berisi judul, deskripsi, prioritas, dan status.

## 4. Kebutuhan Non-Fungsional

### 4.1 Usability

Aplikasi harus mudah digunakan oleh pengguna baru dengan tampilan sederhana dan navigasi yang jelas.

### 4.2 Performance

Aplikasi harus berjalan ringan dan responsif pada perangkat Android.

### 4.3 Reliability

Aplikasi harus dapat menjalankan fitur CRUD tanpa error selama penggunaan normal.

### 4.4 Maintainability

Kode program harus disusun rapi menggunakan konsep class dan object agar mudah dikembangkan.

### 4.5 Portability

Aplikasi dikembangkan menggunakan Flutter sehingga dapat dikembangkan lebih lanjut untuk platform lain.

## 5. Perancangan Sistem

### 5.1 Aktor

Aktor utama dalam aplikasi adalah pengguna.

### 5.2 Use Case

Use case aplikasi:

1. Pengguna membuka aplikasi.
2. Pengguna melihat daftar tugas.
3. Pengguna menambahkan tugas.
4. Pengguna mengubah tugas.
5. Pengguna menghapus tugas.
6. Pengguna menandai tugas selesai.
7. Pengguna melihat detail tugas.

### 5.3 Struktur Class

#### 1. Class Task

Class Task digunakan untuk merepresentasikan data tugas umum.

Atribut:

1. title
2. description
3. isCompleted

Method:

1. markAsDone()
2. markAsUndone()
3. getTaskInfo()

#### 2. Class PriorityTask

Class PriorityTask merupakan turunan dari class Task yang memiliki tambahan atribut priority.

Atribut:

1. priority

Method:

1. getTaskInfo()

Class ini menerapkan inheritance dan method overriding.

#### 3. Class TaskManager

Class TaskManager digunakan untuk mengelola kumpulan object Task.

Atribut:

1. tasks

Method:

1. addTask()
2. updateTask()
3. deleteTask()
4. getAllTasks()

#### 4. Class User

Class User digunakan untuk merepresentasikan pengguna aplikasi.

Atribut:

1. name
2. taskManager

Method:

1. createTask()
2. viewTasks()

Class User memiliki relasi dengan TaskManager.

## 6. Implementasi Konsep OOP

### 6.1 Class dan Object

Aplikasi menggunakan beberapa class seperti Task, PriorityTask, TaskManager, dan User. Object dibuat dari class tersebut untuk merepresentasikan data nyata pada aplikasi.

### 6.2 Constructor

Constructor digunakan untuk memberikan nilai awal pada object, misalnya saat membuat tugas baru.

### 6.3 Encapsulation

Atribut pada class dapat dibuat private dan diakses melalui getter dan setter agar data lebih aman.

### 6.4 Inheritance

Class PriorityTask mewarisi atribut dan method dari class Task.

### 6.5 Method Overriding

Class PriorityTask melakukan override terhadap method getTaskInfo() untuk menampilkan informasi tugas beserta prioritasnya.

### 6.6 Polymorphism

Object PriorityTask dapat diperlakukan sebagai object Task sehingga aplikasi dapat menampilkan berbagai jenis tugas dengan method yang sama.

### 6.7 Relasi Antar Class

Class User memiliki relasi dengan TaskManager, sedangkan TaskManager memiliki relasi dengan kumpulan object Task.

## 7. Rancangan Antarmuka

### 7.1 Halaman Utama

Halaman utama menampilkan daftar tugas pengguna dengan tampilan card berwarna putih dan aksen biru.

Komponen:

1. AppBar dengan nama aplikasi “Tugasku”.
2. Daftar tugas.
3. Tombol tambah tugas.
4. Checkbox status tugas.
5. Tombol edit dan hapus.

### 7.2 Halaman Tambah Tugas

Halaman ini digunakan untuk menambahkan tugas baru.

Komponen:

1. Input judul tugas.
2. Input deskripsi tugas.
3. Pilihan prioritas.
4. Tombol simpan.

### 7.3 Halaman Edit Tugas

Halaman ini digunakan untuk mengubah data tugas yang sudah ada.

Komponen:

1. Input judul tugas.
2. Input deskripsi tugas.
3. Pilihan prioritas.
4. Tombol update.

### 7.4 Halaman Detail Tugas

Halaman ini menampilkan informasi lengkap dari tugas yang dipilih.

## 10. Kesimpulan

Aplikasi “Tugasku” merupakan aplikasi To-Do List berbasis Flutter yang dirancang untuk membantu pengguna mengelola tugas harian. Aplikasi ini memiliki fitur CRUD sederhana dan menerapkan konsep Object Oriented Programming seperti class, object, constructor, encapsulation, inheritance, method overriding, polymorphism, dan relasi antar class.
